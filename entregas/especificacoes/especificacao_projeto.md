# Especificação do Projeto de Banco de Dados
Universidade de Brasília Instituto de Ciências Exatas Departamento de Ciência da Computação Disciplina: Banco de Dados
## 1. Visão Geral
Este documento reúne as especificações do projeto de Banco de Dados para a disciplina de Banco de Dados da Universidade de Brasília. O tema escolhido deve apoiar atividades acadêmicas na UnB e pode atender professores, alunos, reitoria, comunicação com a sociedade ou outros públicos relacionados.

## 2. Requisitos Gerais
- Projeto em grupo de 2 até 4 estudantes.
- Apresentação obrigatória de todos os componentes do grupo.
- Apenas projetos em funcionamento terão nota contabilizada.
- Pode utilizar ferramentas de IA, mas deve ser informado onde e com qual objetivo.
- Deve conter no mínimo 10 entidades.
- Cada tabela deve ter no mínimo 5 registros.
- CRUD funcionando para o sistema.
- Qualquer SGBD relacional pode ser usado.
- Qualquer linguagem de programação pode ser usada para o CRUD.
- Deve haver interface gráfica para permitir que um usuário leigo acesse o CRUD.

## 3. Tema do Projeto
Qualquer banco de dados que possa ser utilizado para facilitar a vida acadêmica na UnB.

Exemplos de aplicações possíveis:
- Auxílio para professores
- Gestão de alunos
- Apoio à reitoria
- Comunicação com a sociedade
- Serviços acadêmicos e administrativos

## 4. Datas Importantes
- Envio dos nomes dos componentes do grupo e tema do projeto: 28/04 e 05/05
- Entrega da primeira parte do projeto: 30/05
- Entrega/apresentação final do projeto: 30/06 a 02/07

## 5. Entrega da Primeira Parte do Projeto
A primeira entrega deve incluir:
- Introdução
- Modelo de Entidade-Relacionamento (usando ferramenta de modelagem)
- Modelo Relacional (usando ferramenta de modelagem)
- Script SQL que gera o banco de dados
- Indicação do uso de IA

## 6. Entrega Final do Projeto
A entrega final deve incluir:
- Introdução
- Modelo de Entidade-Relacionamento (usando ferramenta de modelagem)
- Modelo Relacional (usando ferramenta de modelagem)
- Script SQL que gera o banco de dados
- Uso de pelo menos uma tabela com gerador automático de chave primária do SGBD selecionado
- Construção da camada de persistência
  - Enviar link do GitHub com códigos-fonte
  - Enviar diagrama mostrando como a interface gráfica acessa a camada de persistência
- Programa com funções de CRUD (Create, Read, Update, Delete)
- CRUD que acesse mais de uma tabela nas mesmas funções para garantir integridade referencial
- Utilização de pelo menos uma View
- Utilização de pelo menos uma Procedure
- Utilização de pelo menos um Trigger
- Inserção de dado binário no banco (foto, PDF ou outro arquivo)
- Indicação do uso de IA

## 7. Formalidades da Entrega Final
- Documentação organizada e entregue em arquivo único PDF para a primeira entrega.
- Itens obrigatórios para o PDF da primeira entrega:
  - Introdução
  - Diagrama de Entidade-Relacionamento
  - Modelo Relacional
  - Indicação do uso de IA

## 8. Estrutura recomendada do projeto
1. Documentação e especificações
2. Modelagem ER e modelo relacional
3. Script SQL de criação do banco
4. Implementação da camada de persistência
5. Interface gráfica com CRUD
6. Componentes de banco adicionais:
   - View
   - Procedure
   - Trigger
   - Dado binário

## 9. Observações sobre IA
Quando a equipe utilizar IA, deve informar:
- Qual ferramenta de IA foi utilizada
- Em que etapa do projeto a IA foi usada
- Qual objetivo ou problema foi resolvido com IA
